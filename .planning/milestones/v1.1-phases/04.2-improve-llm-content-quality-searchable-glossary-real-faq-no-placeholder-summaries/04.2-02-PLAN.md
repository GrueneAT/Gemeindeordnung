  <name>Task 1: Add glossary filter input HTML and client-side JS filtering</name>
  <files>scripts/generate-pages.js, src/js/main.js</files>
  <behavior>
    - E2E: Glossary page has input#glossar-filter visible
    - E2E: Typing "Befan" in filter shows only terms containing "Befan" in name or definition
    - E2E: Letter sections with no matching terms are hidden (display:none)
    - E2E: Clearing filter input shows all terms again
    - E2E: Filter input has data-pagefind-ignore to prevent search indexing
  </behavior>
  <action>
1. In `scripts/generate-pages.js`, modify `generateGlossaryPage()` to add a filter input BEFORE the A-Z nav bar (around line 700, after the disclaimer div):

```html
<div class="mb-6" data-pagefind-ignore>
  <input id="glossar-filter" type="search"
    placeholder="Begriff suchen..."
    class="w-full sm:w-96 border border-gray-300 rounded-lg px-4 py-2 text-sm
           bg-white text-gruene-dark focus:outline-none focus:ring-2
           focus:ring-gruene-green/50 focus:border-gruene-green"
    aria-label="Glossarbegriffe filtern" />
</div>
```

2. In `src/js/main.js`, add an `initGlossaryFilter()` function:
   - Get input#glossar-filter; return early if not on glossary page
   - On 'input' event, get lowercased query
   - Select all `.mb-8` containers (letter groups)
   - Within each, find the h2 letter heading and all term divs (children with id attribute that are not the h2)
   - For each term div: check if h3 text or p text includes the query; toggle display:none
   - Track if any term in the letter group is visible; hide the entire .mb-8 if none visible
   - When query is empty, show everything

3. Call `initGlossaryFilter()` in the DOMContentLoaded handler alongside existing init calls.

4. Use the EXACT same Tailwind classes and patterns from the research document. The filter is purely visual (display toggle) -- Pagefind indexes at build time so runtime hiding is safe.
  </action>
  <verify>
    <automated>npm run build && npx pagefind --site dist --force-language de && npx playwright test --config=e2e/playwright.config.js --project=desktop-chromium e2e/tests/glossar.spec.js</automated>
  </verify>
  <done>Glossary page shows filter input, typing filters terms instantly, clearing shows all, letter sections hide when empty, E2E tests pass</done>
</task>
