# Phase 04.3: Curated FAQ Topic List & Fix Broken FAQ Generation - Context

**Gathered:** 2026-03-12
**Status:** Ready for planning

<domain>
## Phase Boundary

Build a curated, research-driven FAQ topic list that persists across regeneration cycles. Fix the broken FAQ generation pipeline (JSON truncation from output token limits). Regenerate FAQ content using the curated topic list. No new UI features — pipeline and data quality only.

</domain>

<decisions>
## Implementation Decisions

### Root Cause: FAQ Generation Failure
- The FAQ prompt sends ~200KB of topic taxonomy to Claude, producing a response that exceeds the 32K output token limit
- Response is truncated mid-JSON — starts with a closing `}` fragment, so even `repairJSON()` cannot recover
- Debug evidence: `_debug_raw_stdout.txt` shows `maxOutputTokens: 32000` hit with `outputTokens: 34923` (attempted)
- Fix approach: Split FAQ generation into multiple smaller LLM calls (topic-by-topic or batch-of-topics), each well within token limits

### Curated Topic List
- Create `data/llm/faq/curated-topics.json` as the authoritative topic list
- Derive initial topics from: existing summary topic tags (collectTopicTaxonomy output), the partial FAQ debug output, and municipal governance domain knowledge
- Structure per topic: `{ slug, title, description, seedQuestions: [string] }` — seed questions guide LLM but are not binding
- Target: 12-18 curated topics covering key municipal governance themes (Sitzungsoeffentlichkeit, Befangenheit, Buergermeisterwahl, Gemeindevorstand, Gemeindehaushalt, Pruefungsausschuss, etc.)
- The curated list is version-controlled and human-editable — it survives regeneration cycles

### FAQ Generation Strategy
- Generate FAQ content per-topic (one LLM call per curated topic) instead of one massive call for all topics
- Each call receives: the curated topic metadata + relevant paragraph references from summaries (filtered to that topic)
- Keep prompt size manageable — include only paragraphs tagged with the target topic
- LLM can add follow-up questions beyond seed questions but must stay within the topic scope
- Aggregate per-topic results into the final `topics.json`

### JSON Robustness
- Keep existing `repairJSON()` for unescaped quote issues (still needed for summary/glossary calls)
- Add response size validation: warn if response approaches token limits
- Per-topic calls naturally avoid truncation (each topic produces ~2-4KB of JSON, well within limits)

### Regression Safety
- FAQ pipeline changes must not affect `generateSummary()` or `generateGlossary()` code paths
- The shared `callClaude()` function stays unchanged — only `generateFAQ()` is refactored
- Verify with `npm run llm:summaries -- --dry-run` and `npm run llm:glossary -- --dry-run` (or targeted test) that those pipelines still parse correctly

### Claude's Discretion
- Exact curated topic selection (within the domain themes listed above)
- Number of questions per topic
- How to batch topic taxonomy data per LLM call (single topic vs. small batches)
- Whether to add a `--topic` flag for regenerating a single FAQ topic
- Internal refactoring of `generateFAQ()` function structure

</decisions>
