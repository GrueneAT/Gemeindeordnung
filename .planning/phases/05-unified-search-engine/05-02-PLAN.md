<task type="auto">
  <name>Task 1: Implement two-pass search and grouped result rendering in search.js</name>
  <files>src/js/search.js</files>
  <action>
Refactor `src/js/search.js` to implement unified search with two-pass Pagefind queries and grouped result rendering. This is a significant refactor of `executeSearch()`, `handleSearchInput()`, `triggerSearch()`, `renderResults()`, and new rendering functions.

**A. Replace `executeSearch()` with `executeUnifiedSearch()`:**

Implement manual debounce at UI level (setTimeout/clearTimeout on input handler), then use `pf.search()` (NOT `debouncedSearch`) for both passes in Promise.all. This avoids the Pagefind debounce conflict (research Pitfall 1).

When BL filter is active (not null):
- Pass 1: `pf.search(query, { filters: { typ: "Gesetz", bundesland: activeBundesland } })`
- Pass 2: `pf.search(query, { filters: { typ: ["FAQ", "Glossar"] } })`
- Run both in `Promise.all`

When BL filter is "Alle" (null):
- Single pass: `pf.search(query, {})`
- Group results client-side by `result.filters.typ[0]`

Load result data: For Gesetze, load first 5 results. For FAQ and Glossar, load ALL results (there are far fewer). Return structured object:
```javascript
{
  faq: { results: [...loadedData], totalCount: N },
  glossar: { results: [...loadedData], totalCount: N },
  gesetz: { results: [...loadedData], totalCount: N, hasMore: bool, allResults: [...rawResults] }
}
```

**B. Add new rendering functions:**

`renderUnifiedResults(searchResult, query)` — Main renderer that builds grouped HTML:
1. FAQ section first (if any): group heading "FAQ Antworten (N)" with badge, then FAQ result cards
2. Glossar section (if any): group heading "Glossar (N)" with badge, then Glossar result cards
3. Gesetze section (if any): group heading "Paragraphen (N)" with badge, then existing Gesetze rendering
4. If ALL groups empty, call `renderEmptyState()`
5. Groups with zero results are hidden entirely (no heading rendered)

`renderTypeGroupHeading(label, count, type)` — Returns HTML for a group heading with content-type badge:
```html
<div class="search-type-group">
  <div class="search-type-group-heading">
    <span class="search-type-badge search-type-{type}">{label}</span>
    <span class="search-type-group-count">({count})</span>
  </div>
</div>
```

`renderFAQResult(result)` — Renders a single FAQ result card:
- Title: `result.meta.topic_title` (from data-pagefind-meta) or cleaned page title
- Snippet: First ~120 chars of `result.excerpt` (use Pagefind's excerpt which has `<mark>` highlights)
- Link: `result.url`
- Add a subtle "FAQ" badge inline

`renderGlossarResult(result)` — Renders Glossar results:
- If `result.sub_results` exist (Pagefind breaks on h3 headings), render each sub_result as a separate card: term name from `sub.title` (bold), first ~100 chars of `sub.excerpt`, link to `sub.url` (with anchor)
- If no sub_results, render page-level: title "Glossar der Rechtsbegriffe", excerpt from `result.excerpt`, link to `result.url`
- Add a subtle "Glossar" badge inline

**C. Update `handleSearchInput()` and `triggerSearch()`:**

Replace the Pagefind `debouncedSearch` with a manual debounce wrapper:
```javascript
let searchTimer = null;
async function handleSearchInput(e) {
  const query = e.target.value.trim();
  currentQuery = query;
  if (query.length === 0) { hideDropdown(); return; }
  if (query.length < 3) { showMinCharsHint(); return; }
  clearTimeout(searchTimer);
  searchTimer = setTimeout(async () => {
    const result = await executeUnifiedSearch(query, activeBundesland);
    renderUnifiedResults(result, query);
  }, 200);
}
```

**D. Update `renderResults()` → `renderUnifiedResults()`:**

Replace the old `renderResults()` with `renderUnifiedResults()`. The new function:
- Receives the structured `{ faq, glossar, gesetz }` object
- Builds grouped HTML using the three section renderers
- Handles "show all" button for Gesetze section (load remaining results)
- Shows total count across all types at the top: "N Treffer" (sum of all groups)

**E. Add BL filter note:**

When `activeBundesland` is set, add a subtle note near the Gesetze group heading or below the filter chips:
```html
<div class="search-filter-note">Filter gilt nur fuer Gesetzestexte</div>
```
This note is hidden when BL filter is "Alle".

**F. Preserve existing behavior:**

- `renderFilterChips()` — no changes needed
- `setupKeyboardShortcuts()` — no changes needed
- `setupMobileSearch()` / `openMobileOverlay()` / `closeMobileOverlay()` — no changes needed; they swap `searchDropdown` ref, and new rendering writes to `searchDropdown` as before
- `setupClickOutside()` — no changes needed
- `initSearch()` — no changes needed
- Export list — update if function names changed (keep `renderResults` export pointing to new unified function for backward compatibility, or rename)

**G. Update "show all" button text:**

Change from "Alle N Gesetze durchsuchen" to "Alle N Paragraphen anzeigen" since results now include non-Gesetze content.
