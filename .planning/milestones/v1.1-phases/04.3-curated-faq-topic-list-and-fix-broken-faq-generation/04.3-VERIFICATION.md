### Gaps Summary

**Gap 1 (Partial): llm:validate exit code**

`npm run llm:validate` exits with code 1 due to 21 pre-existing summary quality issues (short summaries in burgenland, kaernten, oberoesterreich, wien). None of these are FAQ validation errors — `grep '\[FAQ\]'` on the output returns nothing. The FAQ content itself passes all validateFAQ() checks.

The plan's success criterion states "Generated FAQ data passes npm run llm:validate" but the overall command fails due to summary issues that pre-date Phase 04.3. These are documented in `deferred-items.md`. Resolution options: (a) fix the 21 summary issues by regenerating short summaries, or (b) accept that the success criterion should be interpreted as "no FAQ validation errors" rather than the full exit code.

**Gap 2 (Documentation): ROADMAP checkbox**

`ROADMAP.md` shows `- [ ] 04.3-02-PLAN.md` (unchecked) despite STATE.md confirming completion and 04.3-02-SUMMARY.md existing with commit `0ade91b`. This is a documentation inconsistency only — all functional artifacts are present and verified.

**Notable (not blocking): Stale FAQ pages**

`src/faq/` contains 20 old pre-curated HTML pages (e.g. `befangenheit.html`, `buergermeister.html`) that were generated in Phase 4.2. `generate-pages.js` adds new curated pages but does not clean the directory first. These stale pages get copied into `dist/faq/` by the Vite build, alongside the correct new curated pages. This does not break the curated FAQ goal but creates orphaned HTML files in the site.
