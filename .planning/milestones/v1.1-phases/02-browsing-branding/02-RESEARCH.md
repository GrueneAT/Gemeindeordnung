# Phase 2: Browsing & Branding - Research

**Researched:** 2026-03-11
**Domain:** Static site UI/UX, accessibility, Gruenes CI branding
**Confidence:** HIGH

## Summary

Phase 2 transforms the existing basic HTML law pages (generated by `scripts/generate-pages.js`) into a polished, branded browsing experience with auto-generated table of contents, deep links, readable typography, responsive layout, and Gruenes Corporate Identity. All changes flow through the page generator -- this is a static site with no client-side framework, just vanilla JS for interactions (ToC collapse, clipboard copy, scroll-to-top).

The primary technical challenge is WCAG 2.1 AA compliance with the Gruene color palette. The existing `gruene-green` (#6BA539) **fails** contrast requirements (2.97:1 vs white, needs 4.5:1). The Bildgenerator's `gruene-dark` (#2D5016) passes at 9.25:1 and the project's `gruene-dark` (#005538) passes at 8.91:1. Links and interactive text must use darker greens; the bright green is only safe for large decorative elements.

**Primary recommendation:** Refactor `generate-pages.js` to emit the enhanced HTML structure (ToC, breadcrumbs, header/footer, paragraph anchors), add a small vanilla JS module for interactions (ToC toggle, clipboard copy, scroll-to-top, anchor highlight), and extend the TailwindCSS theme with a WCAG-compliant Gruene color palette.

### Pattern 1: Page Generator as Single Source of Truth
**What:** All HTML structure flows through `generate-pages.js`. No manual HTML editing of law pages.
**When to use:** Always -- the generator reads parsed JSON and produces complete HTML pages.
**Example:**
```javascript
// generate-pages.js produces the full page including ToC, header, footer
function generateLawPage(law, key, category, allLaws) {
  const toc = generateTableOfContents(law.struktur);
  const header = generateHeader(law, allLaws); // includes Bundesland dropdown
  const breadcrumb = generateBreadcrumb(law, category);
  const footer = generateFooter(law);
  const content = renderStructuredContent(law.struktur); // paragraphs with #p{nr} anchors
  // ... assemble full HTML
}
```

### Pattern 2: Progressive Enhancement with Vanilla JS
**What:** Pages are fully functional without JavaScript. JS adds enhancements (copy buttons, scroll-to-top, ToC interactivity beyond native `<details>`).
**When to use:** For all interactive features.
**Key principle:** The page must be browsable with JS disabled. ToC uses native `<details>` elements that work without JS. JS adds copy-link icons and smooth scroll behavior.

### Pattern 3: CSS-Only Anchor Highlighting
**What:** Use CSS `:target` pseudo-class combined with a CSS animation for the green highlight fade.
**When to use:** When user navigates to `#p42`.
**Example:**
```css
/* Highlight targeted paragraph */
article:target {
  animation: anchor-highlight 2s ease-out;
}
@keyframes anchor-highlight {
  from { background-color: var(--color-gruene-light); }
  to { background-color: transparent; }
}
```

### Pattern 4: Responsive Mobile-First Design
**What:** Design for mobile first (320px+), enhance for tablet (768px+) and desktop (1024px+).
**When to use:** All layout decisions.
**Key breakpoints:**
- Mobile (<768px): Full-width content, stacked cards, hamburger or simplified header
- Tablet (768px+): 2-column card grid, full header
- Desktop (1024px+): 3-column card grid, max-width container

### Anti-Patterns to Avoid
- **Custom accordion JS for collapsible sections:** Use native `<details>`/`<summary>` -- they are keyboard-accessible, screen-reader-friendly, and require zero JS.
- **Bright green (#6BA539 or #8AB414) for body text or links:** Fails WCAG AA contrast. Use only for large decorative elements or backgrounds.
- **Fixed-position sidebar ToC on mobile:** Takes too much screen space on mobile. Use a sticky collapsible ToC at the top instead.
- **Inline styles in generated HTML:** Keep all styling in TailwindCSS classes for consistency and maintainability.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Collapsible sections | Custom JS accordion | HTML `<details>`/`<summary>` | Built-in accessibility, keyboard support, no JS needed |
| Clipboard copy | Custom clipboard fallback | `navigator.clipboard.writeText()` | Baseline browser support since March 2025, HTTPS required (GitHub Pages provides this) |
| Scroll-to-top button | Complex scroll detection | `window.scrollY > 300` + `window.scrollTo({top: 0, behavior: 'smooth'})` | Simple threshold check, native smooth scroll |
| Responsive grid | Manual media queries | TailwindCSS responsive prefixes (`sm:`, `md:`, `lg:`) | Already in project, utilities handle all breakpoints |
| Anchor scroll offset | Manual scroll calculation | `scroll-margin-top` CSS property | Native CSS, accounts for sticky header |

**Key insight:** This phase involves zero complex client-side logic. Everything is either a page generator enhancement or a few lines of vanilla JS. Resist the temptation to introduce a JS framework.

## Common Pitfalls

### Pitfall 1: Bright Green Text Fails WCAG AA
**What goes wrong:** Using #6BA539 or #8AB414 for body text or links results in insufficient contrast (< 4.5:1 against white/light backgrounds).
**Why it happens:** The Gruene brand green is designed for large visual elements (posters, banners), not body text.
**How to avoid:** Use #005538 (gruene-dark) for all text. Reserve bright green for decorative accents, large headings (24px+ where 3:1 suffices), and backgrounds.
**Warning signs:** Any text using `text-gruene-green` class on a light background.

### Pitfall 2: Paragraph ID Collision with URL Hash
**What goes wrong:** Current IDs use `par-{nummer}` but the user decision specifies `#p42` format. Inconsistency breaks deep links.
**Why it happens:** Phase 1 used `id="par-{nummer}"` pattern.
**How to avoid:** Change all paragraph IDs to `id="p{nummer}"` in the generator. Update all internal links. This is a breaking change from Phase 1 output but no external consumers exist yet.
**Warning signs:** Any anchor link using `#par-` prefix instead of `#p`.

### Pitfall 3: Sticky Header Covering Anchor Targets
**What goes wrong:** Navigating to `#p42` scrolls the paragraph behind the sticky header.
**Why it happens:** Default browser anchor scrolling does not account for fixed/sticky elements.
**How to avoid:** Add `scroll-margin-top` to paragraph articles matching the header height (e.g., `scroll-margin-top: 5rem`).
**Warning signs:** Clicking a ToC link and not seeing the paragraph heading.

### Pitfall 4: Clipboard API Requires HTTPS
**What goes wrong:** `navigator.clipboard.writeText()` fails silently on HTTP.
**Why it happens:** Clipboard API requires a secure context.
**How to avoid:** GitHub Pages serves HTTPS by default. For local dev, Vite dev server runs on localhost (which counts as secure). No action needed, but be aware during testing.
**Warning signs:** Copy button does nothing in local dev on non-localhost URL.

### Pitfall 5: ToC Overwhelming on Laws with Many Sections
**What goes wrong:** A law with 15+ Hauptstuecke and 50+ Abschnitte creates a very long ToC.
**Why it happens:** Some Gemeindeordnungen are extensive (e.g., Wiener Stadtverfassung).
**How to avoid:** Default to collapsed state for all sections. Use `<details>` without `open` attribute. Keep ToC compact with Hauptstueck names only at top level, Abschnitte as nested items.
**Warning signs:** ToC taller than 80vh on desktop.

### Pitfall 6: Mobile Touch Targets Too Small
**What goes wrong:** Copy-link icons and ToC expand buttons are hard to tap on mobile.
**Why it happens:** Designing for desktop hover interactions first.
**How to avoid:** Minimum 44x44px touch targets (WCAG 2.5.5). On mobile, show copy icons always (not just on hover). Use adequate padding on `<summary>` elements.
**Warning signs:** Touch targets smaller than 44px.

## Code Examples

### ToC Generation from Parsed Structure
```javascript
// Source: Based on existing law.struktur JSON schema
function generateTableOfContents(struktur) {
  let toc = '<nav aria-label="Inhaltsverzeichnis" class="mb-8">\n';
  toc += '<h2 class="text-xl font-bold text-gruene-dark mb-4">Inhaltsverzeichnis</h2>\n';

  for (const section of struktur) {
    toc += '<details class="mb-2">\n';
    toc += `  <summary class="cursor-pointer py-2 px-3 rounded hover:bg-gruene-light font-medium text-gruene-dark">`;
    toc += `${escapeHtml(section.nummer)}. ${section.typ === 'hauptstueck' ? 'Hauptstueck' : 'Abschnitt'}: ${escapeHtml(section.titel)}`;
    toc += `</summary>\n`;
    toc += '  <ul class="ml-6 mt-1">\n';

    const paragraphen = section.paragraphen || [];
    // Handle nested abschnitte in hauptstuecke
    if (section.abschnitte) {
      for (const abs of section.abschnitte) {
        toc += `    <li class="font-medium mt-2 text-gruene-dark/80">${escapeHtml(abs.nummer)}. Abschnitt: ${escapeHtml(abs.titel)}</li>\n`;
        for (const p of (abs.paragraphen || [])) {
          toc += `    <li class="py-0.5"><a href="#p${escapeHtml(p.nummer)}" class="text-gruene-dark hover:underline">&sect; ${escapeHtml(p.nummer)}${p.titel ? ' ' + escapeHtml(p.titel) : ''}</a></li>\n`;
        }
      }
    } else {
      for (const p of paragraphen) {
        toc += `    <li class="py-0.5"><a href="#p${escapeHtml(p.nummer)}" class="text-gruene-dark hover:underline">&sect; ${escapeHtml(p.nummer)}${p.titel ? ' ' + escapeHtml(p.titel) : ''}</a></li>\n`;
      }
    }

    toc += '  </ul>\n</details>\n';
  }

  toc += '</nav>';
  return toc;
}
```

### Clipboard Copy with Tooltip Feedback
```javascript
// Source: MDN Clipboard API documentation
function initCopyLinks() {
  document.querySelectorAll('[data-copy-link]').forEach(btn => {
    btn.addEventListener('click', async (e) => {
      e.preventDefault();
      const paragraphId = btn.dataset.copyLink;
      const url = `${window.location.origin}${window.location.pathname}#${paragraphId}`;

      try {
        await navigator.clipboard.writeText(url);
        showTooltip(btn, 'Link kopiert!');
      } catch {
        // Fallback: select+copy for older contexts
        const input = document.createElement('input');
        input.value = url;
        document.body.appendChild(input);
        input.select();
        document.execCommand('copy');
        document.body.removeChild(input);
        showTooltip(btn, 'Link kopiert!');
      }
    });
  });
}

function showTooltip(anchor, text) {
  const tooltip = document.createElement('span');
  tooltip.textContent = text;
  tooltip.className = 'absolute -top-8 left-1/2 -translate-x-1/2 bg-gruene-dark text-white text-xs px-2 py-1 rounded whitespace-nowrap';
  anchor.style.position = 'relative';
  anchor.appendChild(tooltip);
  setTimeout(() => tooltip.remove(), 2000);
}
```

### Scroll-to-Top Button
```javascript
function initScrollToTop() {
  const btn = document.getElementById('scroll-to-top');
  if (!btn) return;

  window.addEventListener('scroll', () => {
    btn.classList.toggle('hidden', window.scrollY < 300);
  }, { passive: true });

  btn.addEventListener('click', () => {
    window.scrollTo({ top: 0, behavior: 'smooth' });
  });
}
```

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| Custom JS accordions | `<details>`/`<summary>` elements | Baseline since 2020 | Zero JS for basic collapse, built-in accessibility |
| `document.execCommand('copy')` | `navigator.clipboard.writeText()` | Baseline March 2025 | Promise-based, cleaner API, requires HTTPS |
| Manual scroll detection | `IntersectionObserver` for scroll-aware UI | Baseline 2020 | More performant than scroll listeners (but simple scrollY check is fine for scroll-to-top) |
| CSS media queries | TailwindCSS responsive prefixes | TailwindCSS v3+ | Utility-first responsive design |

**Deprecated/outdated:**
- `document.execCommand('copy')`: Deprecated but still works. Use as fallback only for edge cases.

## Open Questions

1. **Gruene Logo Source**
   - What we know: Logo exists as ~820x820px PNG/JPG and SVG on gruene.at/Bildgenerator
   - What's unclear: Exact SVG file URL for download; licensing terms for party-affiliated tool
   - Recommendation: Download logo from Bildgenerator GitHub repo or gruene.at. Store as `src/assets/gruene-logo.svg`. If not available as SVG, use optimized PNG.

2. **Paragraph Numbering Edge Cases**
   - What we know: Most paragraphs use numeric `nummer` (1, 2, 3...). Some may use alphanumeric (e.g., "2a").
   - What's unclear: Whether all parsed JSON `nummer` fields are URL-safe for anchors
   - Recommendation: URL-encode paragraph IDs in anchors. Sanitize `nummer` to be valid HTML ID (no spaces, no special chars besides letters/numbers/hyphens).
