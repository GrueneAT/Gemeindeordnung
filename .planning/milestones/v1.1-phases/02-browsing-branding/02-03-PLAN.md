<objective>
Fix copy-link button visibility bug that blocks BROW-04 (deep link sharing).

Purpose: Copy buttons are permanently invisible on desktop because the parent article element lacks the Tailwind `group` class needed by `group-hover:opacity-100`. Additionally, the `.copy-link-btn` CSS rule (which handles mobile-always-visible via `@media (hover: hover)`) is orphaned — buttons use inline Tailwind opacity classes instead. This plan reconciles both approaches by using the CSS class.

Output: Working copy-link buttons visible on hover (desktop) and always visible (mobile/touch), unblocking the tooltip feedback flow.
</objective>
