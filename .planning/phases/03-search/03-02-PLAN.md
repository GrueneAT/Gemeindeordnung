**A. Modify `scripts/generate-pages.js` -- Add search HTML to header:**

1. In `generateHeader()`, add a search container between the logo link and the Bundesland dropdown:
   - Desktop: visible search input with placeholder "Suche... (Ctrl+K)" and a search icon
   - Mobile: a search icon button that triggers fullscreen overlay
   - The search input should have `id="search-input"`, `type="search"`, `minlength="3"`, `autocomplete="off"`
   - Below the search input (inside the header container): a div `id="search-chips"` for filter chips (populated by JS)
   - Below that: a div `id="search-dropdown"` for results dropdown (populated by JS, initially hidden)

2. The search HTML must appear on BOTH law pages (isLawPage=true) AND the index page (isLawPage=false). Per CONTEXT.md: "Suchfeld im Header auf jeder Seite".

3. On index page: since there's no Bundesland dropdown, the search field fills that space. On law pages: search field sits between logo and dropdown.

4. Regenerate pages after template changes: `node scripts/generate-pages.js`

**B. Expand `src/js/search.js` -- Add full UI rendering:**

Add the following UI functions to the existing search module (which already has loadPagefind, executeSearch, etc. from Plan 01):

1. `initSearch()` -- Main init function called from main.js:
   - Get DOM refs: `#search-input`, `#search-dropdown`, `#search-chips`, `#search-toggle` (mobile icon)
   - Call `loadPagefind()` eagerly to pre-load WASM
   - Restore saved Bundesland from LocalStorage
   - Render initial filter chips
   - Set up event listeners

2. `handleSearchInput(event)` -- Live search handler:
   - Read query from input
   - If < 3 chars: show hint "Bitte mindestens 3 Zeichen eingeben" in dropdown
   - If >= 3 chars: call `executeSearch(query, activeBundesland)`
   - If result is null (superseded): do nothing
   - Otherwise: call `renderResults(result)`

3. `renderResults({totalCount, results, hasMore})`:
   - Show dropdown (remove hidden class)
   - Header: result count -- "X Treffer" or "X Treffer in [BL]"
   - If totalCount === 0: render empty state (see below)
   - For each result: render a clickable item with:
     - Law title (from meta.title, cleaned -- remove " - Gemeindeordnung.at" suffix)
     - Excerpt with highlighted terms (result.excerpt, safe innerHTML)
     - Link: result.url + "?highlight=" + encodeURIComponent(query)
   - If grouped by BL (when "Alle Bundeslaender" active): group results by result.filters.bundesland, show BL group headings with per-BL count
   - If hasMore: show "Alle X Treffer anzeigen" link that removes the max-height constraint on the dropdown to show all results (load remaining via allResults)

4. `renderEmptyState(query, bundesland)`:
   - "Keine Treffer fuer '[query]' in [BL/Alle Bundeslaender]"
   - Suggestions: "Versuchen Sie einen anderen Suchbegriff"
   - If BL filtered: "In allen Bundeslaendern suchen" (clickable, switches filter)
   - "Zur Uebersicht" link to index page

5. `renderFilterChips(activeBundesland)`:
   - If a Bundesland is saved: show chip "[BL-Name]" (active/highlighted) + "Alle Bundeslaender" chip
   - If no BL saved: show only "Alle Bundeslaender" chip (active)
   - Clicking a chip: updates activeBundesland, saves to LocalStorage, re-runs current search if query exists

6. `setupKeyboardShortcut()`:
   - Listen for "/" key (when not in input/textarea) or Ctrl+K: focus search input, prevent default
   - Listen for Escape: close dropdown, blur search input

7. `setupMobileSearch()`:
   - On mobile (< 640px): the search icon button shows/hides a fullscreen overlay
   - The overlay contains the search input (moved or cloned), filter chips, and results
   - Backdrop overlay prevents interaction with page behind
   - Close button or Escape dismisses

8. `setupClickOutside()`:
   - Clicking outside the search dropdown closes it
