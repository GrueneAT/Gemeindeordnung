<task type="auto">
  <name>Task 2: Create GitHub Actions deployment workflow</name>
  <files>.github/workflows/deploy.yml, .nojekyll</files>
  <action>
1. Create `.github/workflows/deploy.yml` with manual trigger only (workflow_dispatch -- user decision: no automatic schedule):
   ```yaml
   name: Deploy to GitHub Pages
   on:
     workflow_dispatch:
   permissions:
     contents: read
     pages: write
     id-token: write
   concurrency:
     group: 'pages'
     cancel-in-progress: true
   jobs:
     deploy:
       environment:
         name: github-pages
         url: ${{ steps.deployment.outputs.page_url }}
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v5
         - uses: actions/setup-node@v6
           with:
             node-version: 22
             cache: 'npm'
         - run: npm ci
         - name: Fetch laws from RIS
           run: node scripts/fetch-laws.js
         - name: Parse laws into JSON
           run: node scripts/parse-laws.js
         - name: Generate HTML pages
           run: node scripts/generate-pages.js
         - name: Build with Vite
           run: npm run build
         - uses: actions/configure-pages@v5
         - uses: actions/upload-pages-artifact@v4
           with:
             path: './dist'
         - uses: actions/deploy-pages@v4
           id: deployment
   ```

2. Create `.nojekyll` file (empty) at project root to prevent GitHub Pages from processing with Jekyll.

3. The workflow runs the FULL pipeline: fetch -> parse -> generate -> build -> deploy.
   - If any step fails (fetch error, parse error, build error), the entire workflow fails (fail-fast per user decision).
   - LLM content is NOT generated during deploy -- it reads from committed data/llm/ directory (per DEPL-04).

IMPORTANT per user decisions:
- Manual trigger ONLY (workflow_dispatch), no push/schedule triggers
- GitHub Actions Pages direct deployment (no gh-pages branch)
- Build aborts if any Bundesland fails to fetch or parse
