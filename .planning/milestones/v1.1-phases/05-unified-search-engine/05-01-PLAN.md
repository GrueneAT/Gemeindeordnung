<task type="auto">
  <name>Task 1: Add Pagefind typ filter and meta attributes to all page templates</name>
  <files>scripts/generate-pages.js</files>
  <action>
Modify three template functions in `scripts/generate-pages.js`:

**1. `generateLawPage()` (line 439):**
Change the existing typ filter from category slug to "Gesetz":
```javascript
// BEFORE (line 439):
<meta data-pagefind-filter="typ[content]" content="${escapeHtml(category)}" />
// AFTER:
<meta data-pagefind-filter="typ[content]" content="Gesetz" />
```
This is a one-line change. The `category` variable outputs "gemeindeordnungen" or "stadtrechte" — we need "Gesetz" for unified search filtering.

**2. `generateFAQTopicPage()` (around line 620-625):**
Add two meta tags inside the `<head>` section, after the `<link>` to main.css (around line 625):
```html
<meta data-pagefind-filter="typ[content]" content="FAQ" />
<meta data-pagefind-meta="topic_title[content]" content="${genderText(escapeHtml(topic.title))}" />
```
These enable Pagefind to (a) filter FAQ results separately and (b) expose the topic title as metadata for rich result rendering.

**3. `generateGlossaryPage()` (around line 710-713):**
Add one meta tag inside the `<head>` section, after the `<link>` to main.css (around line 713):
```html
<meta data-pagefind-filter="typ[content]" content="Glossar" />
```

**4. `generateFAQIndexPage()` (around line 585):**
The FAQ index page currently has `<main>` WITHOUT `data-pagefind-body`. Verify it stays that way — it should NOT be indexed (it is a navigation page with cards, not content). If it somehow has `data-pagefind-body`, remove it. Also add `data-pagefind-ignore` to the `<main>` tag to be explicit:
```html
<main data-pagefind-ignore>
```

Do NOT add any `typ` filter to the FAQ index page.
