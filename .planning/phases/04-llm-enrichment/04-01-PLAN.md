  <action>
Extend the existing `llm-analyze.js` stub into a full LLM generation pipeline. The script uses Claude Code CLI (`claude` command) via `child_process.execSync` or `spawn` to generate content using the existing subscription (no API costs).

**Generation steps (run sequentially):**

1. **Per-law summaries + topics** (`generateForLaw(lawKey, category, rootDir)`):
   - Read parsed JSON from `data/parsed/{category}/{lawKey}.json`
   - Flatten all paragraphs from `struktur` (handle nested hauptstueck/abschnitte)
   - For each law, invoke Claude CLI with a prompt that provides the full law text and asks for:
     - A `summary` per paragraph: 1-3 sentences, sachlich-verstaendlich tone, starting with "Dieser Paragraph regelt, dass..." or similar
     - `topics` per paragraph: 1-3 topic labels from a consistent taxonomy (e.g., "Gemeinderatssitzungen", "Abstimmungen", "Befangenheit", "Gemeindevorstand", etc.)
   - Output JSON structure per research pattern:
     ```json
     {
       "meta": { "generatedAt": "...", "lawKey": "...", "category": "..." },
       "paragraphs": {
         "1": { "summary": "...", "topics": ["Topic1", "Topic2"] },
         "2": { "summary": "...", "topics": ["Topic1"] }
       }
     }
     ```
   - Write to `data/llm/summaries/{category}/{lawKey}.json`
   - Use incremental skip: if output file already exists, skip (existing logic)

2. **Topic taxonomy extraction** (after all laws processed):
   - Read all generated summary JSONs
   - Collect all unique topic labels across all laws
   - This becomes the unified taxonomy for FAQ generation

3. **FAQ generation** (`generateFAQ(rootDir)`):
   - Read all summary JSONs to understand the full content
   - Invoke Claude CLI with all topic labels and representative paragraph content
   - Ask for 8-15 FAQ topics (Claude determines based on content), each with:
     - `slug`, `title`, `description`
     - 3-6 `questions` per topic, each with `question`, `answer` (mentioning cross-BL differences), and `references` array linking to specific paragraphs
   - References format: `{ "category": "gemeindeordnungen", "key": "burgenland", "paragraph": "42", "label": "Paragraph 42 Bgld. GO" }`
   - Write to `data/llm/faq/topics.json`

4. **Glossary generation** (`generateGlossary(rootDir)`):
   - Invoke Claude CLI with representative law text from multiple Bundeslaender
   - Ask for a conservative list of genuine Fachbegriffe (legal terms a layperson would not know)
   - Examples: Befangenheit, Kollegialorgan, Dringlichkeitsantrag, Geschaeftsordnung, Beschlussfaehigkeit
   - Each term: `{ "term": "...", "slug": "...", "definition": "...", "references": [...] }`
   - Slug: `term.toLowerCase().replace(/[aeoeue]/special).replace(/[^a-z0-9]+/g, '-')`
   - Handle umlauts: ae->ae, oe->oe, ue->ue, ss->ss (keep ASCII for URL slugs)
   - Write to `data/llm/glossary/terms.json`

**Claude CLI invocation pattern:**
```javascript
import { execSync } from 'child_process';

function callClaude(prompt) {
  const result = execSync(
    `echo ${JSON.stringify(prompt)} | claude -p --output-format json`,
    { encoding: 'utf-8', maxBuffer: 10 * 1024 * 1024, timeout: 300000 }
  );
  return JSON.parse(result);
}
```

If `claude` CLI is not available, fall back to creating placeholder JSON with a TODO marker so that downstream plans can still work (the page generator needs JSON files to exist even if content is placeholder).

**CLI entry points:**
- `node scripts/llm-analyze.js --dry-run` — existing, keep working
- `node scripts/llm-analyze.js --generate` — run full pipeline
- `node scripts/llm-analyze.js --generate --law burgenland` — single law only
- `node scripts/llm-analyze.js --faq` — regenerate FAQ only
- `node scripts/llm-analyze.js --glossary` — regenerate glossary only

Keep the existing `dryRun()` export working. Add new exports: `generateForLaw`, `generateFAQ`, `generateGlossary`, `generateAll`.
