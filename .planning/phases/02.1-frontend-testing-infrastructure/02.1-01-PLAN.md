<task type="auto">
  <name>Task 1: Playwright infrastructure setup</name>
  <files>package.json, e2e/playwright.config.js, .gitignore</files>
  <action>
1. Install Playwright and axe-core as dev dependencies:
   ```bash
   npm install -D @playwright/test @axe-core/playwright
   npx playwright install chromium --with-deps
   ```

2. Add npm scripts to package.json:
   - `"test:e2e": "playwright test --config=e2e/playwright.config.js"`
   - `"test:e2e:ui": "playwright test --config=e2e/playwright.config.js --ui"`

3. Create `e2e/playwright.config.js` with:
   - `testDir: './tests'`
   - `outputDir: './test-results'`
   - `fullyParallel: true`
   - `forbidOnly: !!process.env.CI`
   - `retries: process.env.CI ? 1 : 0`
   - `workers: process.env.CI ? 1 : undefined`
   - `use.baseURL: 'http://localhost:4173/gemeindeordnung'` (CRITICAL: must include the /gemeindeordnung/ base path from vite.config.js)
   - `use.screenshot: 'on'` (captures screenshot on every test for TEST-02)
   - `use.trace: 'on-first-retry'`
   - Two projects: `desktop-chromium` using `devices['Desktop Chrome']` and `mobile` using `devices['iPhone 13']` with `viewport: { width: 375, height: 812 }`
   - `webServer.command: 'npm run preview -- --port 4173'`
   - `webServer.url: 'http://localhost:4173/gemeindeordnung/'`
   - `webServer.reuseExistingServer: !process.env.CI`
   - `webServer.timeout: 30000`

4. Add to `.gitignore`:
   ```
   e2e/screenshots/
   e2e/test-results/
   playwright-report/
   ```

IMPORTANT: The site must be built before tests can run. The webServer only serves — it does not build. Before running tests locally, ensure `npm run build` has been run (with generated pages present in src/gemeindeordnungen/ and src/stadtrechte/). The CI workflow handles this automatically via the existing fetch->parse->generate->build pipeline.

**browse-page.spec.js** (TEST-01a):
- Navigate to a Gemeindeordnung page (e.g. `/gemeindeordnungen/wien.html`)
- Assert header is visible with logo (`.gruene-logo`) and site name text
- Assert main content area has law text (article or main element with paragraph content)
- Assert footer is visible with RIS source link and "Keine Rechtsberatung" text
- Assert breadcrumb/heading contains Bundesland name
- Screenshot: `e2e/screenshots/browse-page-wien.png`

**toc.spec.js** (TEST-01b):
- Navigate to a law page
- Find `<details>` elements (collapsible ToC sections)
- Assert at least one details element exists
- Click the `<summary>` to expand — assert `open` attribute appears
- Click again to collapse — assert `open` attribute removed
- Screenshots: `e2e/screenshots/toc-collapsed.png` and `e2e/screenshots/toc-expanded.png`

**card-grid.spec.js** (TEST-01c):
- Navigate to index page (`/index.html` or `/`)
- Assert the Gemeindeordnungen section has a grid with exactly 9 card links
- Assert the Stadtrechte section has a grid with exactly 14 card links
- Assert each card has a Bundesland name (font-bold text), law short title, and Stand datum
- Assert card links have valid href attributes pointing to .html pages
- Screenshot: `e2e/screenshots/card-grid-index.png`

**typography.spec.js** (TEST-01d):
- Navigate to a law page
- Evaluate computed styles on the law text container:
  - `line-height` >= 1.5 (may be in px, convert: lineHeight_px / fontSize_px >= 1.5)
  - `max-width` is set and approximately 70ch (check it's between 600px and 900px, or use ch units)
- Assert text is not cramped: paragraph elements exist with visible text content
- Screenshot: `e2e/screenshots/typography-law-text.png`

**copy-link.spec.js** (TEST-01e):
- Navigate to a law page
- Grant clipboard permissions: `await context.grantPermissions(['clipboard-read', 'clipboard-write'])` — NOTE: must use `context` not `page`. Get context from test fixtures: `test('...', async ({ page, context }) => { ... })`
- Find a `[data-copy-link]` button, click it
- Assert tooltip "Link kopiert!" appears (check for the tooltip span element)
- Assert clipboard contains the expected URL with the paragraph anchor
- Wait for tooltip to disappear (2s timeout + buffer)
- Screenshot: `e2e/screenshots/copy-link-tooltip.png` (captured while tooltip is visible)

IMPORTANT for copy-link: The `.copy-link-btn` buttons use CSS visibility pattern — on desktop they may be visible on hover only. Use `button.dispatchEvent` or `page.locator('[data-copy-link]').first().click({ force: true })` if needed to bypass hover requirement.
