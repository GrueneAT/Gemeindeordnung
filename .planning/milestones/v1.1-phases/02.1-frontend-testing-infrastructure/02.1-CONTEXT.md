# Phase 02.1: Frontend Testing Infrastructure - Context

**Gathered:** 2026-03-11
**Status:** Ready for planning

<domain>
## Phase Boundary

Add Playwright-based end-to-end testing infrastructure that verifies all frontend behaviors (JS interactions + HTML structure + visual appearance) with screenshot capture. Phase 2 delivered 4 interactive behaviors and 10 UAT scenarios — this phase ensures they stay working through automated tests.

</domain>

<decisions>
## Implementation Decisions

### Test scope & strategy
- Playwright E2E tests covering the 10 UAT scenarios from Phase 2 (browse page, ToC, card grid, typography, copy link, scroll-to-top, dropdown nav, anchor highlight, mobile responsive, WCAG contrast)
- Screenshot capture for visual verification — each test takes a screenshot of the relevant state
- Mobile viewport testing included (at least one test runs at 375px width)

### Testing tooling
- Playwright for real browser E2E tests with screenshot capture
- Tests run against the built site (vite build → serve → test)
- Keep existing vitest unit tests for Node.js scripts unchanged

### CI integration
- Add Playwright test step to GitHub Actions workflow before deploy
- Tests must pass before deployment proceeds
- Screenshots saved as CI artifacts for review

### Test data approach
- Build the real site and test against it — most realistic, catches integration issues
- Use a subset of law data if full build is too slow for CI (1-2 Bundesländer sufficient for testing)

### Claude's Discretion
- Exact Playwright configuration and project structure
- Which browsers to test (Chromium-only is fine for CI speed)
- Screenshot naming conventions and storage location
- Whether to use Playwright's built-in test runner or integrate with vitest
- Accessibility assertion approach (axe-playwright or manual checks)
- Test file organization and helper utilities

</decisions>

## Existing Code Insights

### Reusable Assets
- `tests/` directory: 3 existing vitest test files — established test pattern with fixtures
- `src/js/main.js`: 4 functions to test (initCopyLinks, initScrollToTop, initBundeslandDropdown, initAnchorHighlight)
- `scripts/generate-pages.js`: Generates HTML pages that E2E tests will verify

### Established Patterns
- Vitest 4.0.18 for unit tests (keep for scripts, add Playwright alongside)
- ESM modules throughout (`"type": "module"` in package.json)
- Vite 7 build with dynamic multi-page input discovery
- TailwindCSS v4 CSS-first config

### Integration Points
- GitHub Actions workflow needs Playwright test step added before deploy
- `npm run build` produces the testable site output in `dist/`
- Generated HTML in `src/gemeindeordnungen/` and `src/stadtrechte/` — Playwright tests against built output
