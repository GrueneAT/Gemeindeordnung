<decisions>
## Implementation Decisions

### Fix Scope & Standards
- Full polish pass: fix broken layouts, rough edges, AND fine-tune typography/whitespace/visual hierarchy
- Fix autonomously as found: review screenshot → identify issue → fix code → re-screenshot → verify. Iterate until clean.
- Known issue: Logo is wrong — currently sunflower, must be the arrow logo with "Die Grünen" and gruene.at
- Claude identifies remaining issues through systematic screenshot review — no pre-provided list beyond the logo

### Fix Approach
- Build → Pagefind → Playwright screenshot → Read screenshot → Fix → Repeat cycle
- No commit until all screenshots pass visual review (block until clean)
- Silent proceed when screenshots look good — no audit trail paperwork needed

### Workflow Embedding
- Visual review mandatory after every plan execution that touches UI
- Primary mandate lives in CLAUDE.md (strengthen existing Visual Review Protocol to be non-optional)
- Also add project-local GSD references (planning docs, phase templates) as belt-and-suspenders reminder
- Do NOT modify shared GSD templates in /opt/claude-config/ — keep changes project-scoped

### Self-Validation Depth
- Claude validates ALL of: layout correctness, branding accuracy, content/typography, interactive states
- Layout: no overlapping elements, proper spacing, z-index correct, no horizontal overflow, responsive breakpoints
- Branding: correct arrow logo with "Die Grünen", green colors match CI, consistent identity across pages
- Content: umlauts render correctly, readable line-height/max-width, proper contrast, no truncation
- Interactive: tooltips positioned, dropdowns layered, search overlay works, copy-link feedback visible
- Strictness: block commit until ALL screenshots pass — no exceptions

### Review Coverage
- Expand beyond current 22 screenshots where gaps exist (Claude decides what to add during review)
- Viewports: Desktop (1280px) + Mobile (375px) — no tablet or wide desktop
- Bundesland coverage: Wien as single representative page — template works for all BL
- Coverage should capture all page types (index, law page, search states, interactive states)

### Claude's Discretion
- Which additional screenshots to add beyond current 22
- Exact visual fixes (spacing values, color adjustments, typography tweaks)
- How to source/embed the correct arrow logo
- Wording of visual review mandate in CLAUDE.md updates
- Structure of project-local GSD visual review references

</decisions>
