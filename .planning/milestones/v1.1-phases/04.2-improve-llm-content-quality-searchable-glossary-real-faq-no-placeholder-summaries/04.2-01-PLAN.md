  <action>
Create `scripts/llm-validate.js` as both a CLI tool and importable module:

1. Import BL_CITATION from llm-analyze.js for citation format checking.

2. Implement exported validation functions:
   - `validateSummary(data, lawKey)` -- checks a single summary JSON:
     * No `placeholder: true` in meta
     * Each paragraph summary > 50 chars
     * No summary starts with "Dieser Paragraph regelt"
     * No ASCII-safe spellings in summaries (check for "fuer", "ueber", "oesterreich", "aenderung", "aerztlich", "oeffentlich" etc. -- common Austrian German words where ASCII would be wrong)
     * Topic "Allgemeine Bestimmungen" not in >50% of paragraphs
   - `validateFAQ(data)` -- checks FAQ JSON:
     * No `placeholder: true` in meta
     * No formulaic questions matching patterns like "Was regelt das Thema"
     * References use proper citation format (label contains BL_CITATION values, not raw keys)
     * Each answer > 100 chars
   - `validateGlossary(data)` -- checks glossary JSON:
     * No `placeholder: true` in meta
     * Each definition > 30 chars
     * Terms have references spanning at least 3 different BL keys
     * No ASCII-safe spellings in definitions
   - `validateAll(rootDir)` -- runs all validators against files on disk, returns `{ valid: boolean, errors: string[] }`

3. CLI entry point: when run directly, calls validateAll() and prints results. Exit code 0 if valid, 1 if errors found.

4. Create `tests/llm-validate.test.js` with comprehensive unit tests using inline test data (no file I/O needed). Test both passing and failing cases for each validator.
