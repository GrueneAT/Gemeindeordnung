<task type="auto">
  <name>Task 2: GitHub Actions CI integration for Playwright</name>
  <files>.github/workflows/deploy.yml</files>
  <action>
Add Playwright test steps to the existing GitHub Actions deploy workflow. Insert these steps AFTER the "Build with Vite" step and BEFORE the "actions/configure-pages@v5" step:

1. **Install Playwright browsers** step:
   ```yaml
   - name: Install Playwright browsers
     run: npx playwright install chromium --with-deps
   ```

2. **Run E2E tests** step:
   ```yaml
   - name: Run E2E tests
     run: npx playwright test --config=e2e/playwright.config.js
   ```
   This runs ALL tests (both desktop and mobile projects). If any test fails, the workflow fails and deployment is blocked.

3. **Upload test screenshots** step (always runs, even on failure):
   ```yaml
   - name: Upload test screenshots
     if: always()
     uses: actions/upload-artifact@v4
     with:
       name: playwright-screenshots
       path: |
         e2e/screenshots/
         e2e/test-results/
       retention-days: 14
   ```

4. **Upload test report on failure** step:
   ```yaml
   - name: Upload Playwright report
     if: failure()
     uses: actions/upload-artifact@v4
     with:
       name: playwright-report
       path: playwright-report/
       retention-days: 7
   ```

IMPORTANT: The webServer in playwright.config.js runs `npm run preview` which serves the already-built `dist/` directory. The CI workflow already builds the site before these steps, so this works correctly. The webServer config has `reuseExistingServer: !process.env.CI` which means in CI it always starts a fresh server.
