  <name>Task 1: Create unified search E2E tests</name>
  <files>e2e/tests/unified-search.spec.js</files>
  <action>
Create `e2e/tests/unified-search.spec.js` with Playwright tests covering all 5 phase requirements.

**Prerequisites:** Tests must run against the built + Pagefind-indexed site. Use the same test pattern as existing search specs (navigate to index page, type in search input, wait for results).

**Test cases to implement:**

1. **SRCH-02: Cross-type search results** — Search for a term that appears in Gesetze AND FAQ/Glossar content (e.g., "Gemeinderat" which likely appears across all types). Verify that `.search-type-group` elements appear for multiple content types. Assert at least the Gesetze group appears (Gesetze content is guaranteed). If FAQ or Glossar results also appear, verify their group headings.

2. **SRCH-03: Grouped results with badges and counts** — Search and verify:
   - Each visible `.search-type-group` has a `.search-type-badge` element
   - Each visible group has a `.search-type-group-count` with a number in parentheses
   - Group order matches: FAQ before Glossar before Gesetze (check DOM order of `.search-type-group` elements)
   - Groups with zero results are NOT present in the DOM

3. **DSKT-01: Expanded panel dimensions** — After searching, verify the search dropdown:
   - Is visible (not `.hidden`)
   - Has CSS `max-width` of `800px` (computed style check or style assertion)
   - Has CSS `max-height` of approximately `70vh`
   - Capture screenshot: `unified-search-panel.png`

4. **DSKT-02: Rich result context per type** — For Gesetze results, verify `.search-result-title` and `.search-result-excerpt` are present. For FAQ results (if any), verify topic title and snippet text are present. For Glossar results (if any), verify term name is present.

5. **DSKT-03: Space-efficient layout** — Capture screenshot `unified-search-grouped.png` showing grouped results. Visual review will verify scannability.

6. **BL filter behavior** — Select a specific BL filter chip, search, and verify:
   - The filter note "Filter gilt nur fuer Gesetzestexte" appears (`.search-filter-note`)
   - Gesetze results are filtered to that BL
   - FAQ/Glossar sections still appear (if matching results exist)
   - Capture screenshot: `unified-search-bl-filter.png`

7. **Mobile grouped results** — In a 375px viewport test:
   - Open mobile search overlay
   - Search for a term
   - Verify `.search-type-group` elements appear in the mobile overlay
   - Capture screenshot: `unified-search-mobile.png`

**Screenshot list for CLAUDE.md Visual Review Protocol additions:**
- `unified-search-panel.png` — Expanded panel with grouped results
- `unified-search-grouped.png` — Content-type groups with badges and counts
- `unified-search-bl-filter.png` — BL filter active with filter note
- `unified-search-mobile.png` — Mobile overlay with grouped results

Use existing test patterns: `page.goto()`, `page.fill('#search-input', ...)`, `page.waitForSelector('.search-type-group')`, `page.screenshot()`.
