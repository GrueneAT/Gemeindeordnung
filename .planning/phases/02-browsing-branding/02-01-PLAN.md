  <action>
Rewrite `scripts/generate-pages.js` to produce the enhanced HTML structure. Keep the same module API (`generatePages(rootDir)`). All changes are to the HTML output, not the data flow.

**Header (every page):**
- Sticky header with white background, bottom border, z-10
- Left: Gruene logo (img src to ../assets/gruene-logo.svg on law pages, assets/gruene-logo.svg on index), site title "Gemeindeordnung.at"
- Right on law pages: Bundesland dropdown `<select>` with all 23 laws grouped by optgroup (Gemeindeordnungen/Stadtrechte), current law selected. Import LAWS from config.js to populate dropdown options. Use `change` event to navigate (handled by main.js in Plan 02).
- Responsive: stack vertically on mobile, full width

**Breadcrumb (law pages only):**
- `<nav aria-label="Breadcrumb">` with: Startseite > [Bundesland] > [Kurztitel]
- Use text-gruene-dark for links, hover:underline
- On mobile: show abbreviated (just back-arrow + "Uebersicht")

**Table of Contents (law pages only):**
- `<nav aria-label="Inhaltsverzeichnis">`
- `<h2>Inhaltsverzeichnis</h2>`
- For each top-level section in `law.struktur`: `<details>` with `<summary>` showing section type + nummer + titel
- Inside details: `<ul>` with links to each paragraph `<a href="#p{nummer}">` (NOTE: use `p` prefix, not `par-`)
- Handle nested abschnitte inside hauptstuecke (two levels)
- All details elements closed by default (no `open` attribute)
- Style: cursor-pointer summary, padding for touch targets (min 44px height), hover:bg-gruene-light

**Main content (law pages):**
- `<main class="max-w-prose mx-auto leading-relaxed">` for readable typography (70ch max, 1.6 line-height)
- Each top-level section: `<section>` with visual separator (border-t-2 border-gruene-green/20 for hauptstuecke, border-t for abschnitte)
- Hauptstueck headings: text-2xl font-bold text-gruene-dark with mt-12 mb-6
- Abschnitt headings: text-xl font-semibold text-gruene-dark with mt-8 mb-4
- Paragraphs: `<article id="p{nummer}">` (CHANGED from `par-{nummer}` per user decision BROW-03)
- Paragraph heading: `<h3>` with section-sign and nummer, plus `<button data-copy-link="p{nummer}">` for copy icon (Plan 02 wires the JS)
- Paragraph body: keep existing absaetze/text rendering logic but ensure whitespace-pre-line for text

**Footer (every page):**
- Bottom of page, border-t, padding, muted text (text-gray-600)
- Three items: "Quelle: RIS" link to ris.bka.gv.at, Stand datum from meta.fetchedAt (on law pages) or current date (on index), "Keine Rechtsberatung" disclaimer
- GitHub link (to repo, can be placeholder "#" for now)
- Responsive: stack on mobile

**Index page (Startseite):**
- Same header (without Bundesland dropdown)
- Hero section: title + subtitle describing the tool
- Bundesland cards in responsive grid: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`
- Each card: white bg, rounded-lg, shadow-sm, border, hover:shadow-md transition, padding
- Card content: Bundesland name (bold), Kurztitel (text-sm), Stand datum (text-xs text-gray-500)
- Link wraps entire card (a tag with block display)
- Two sections: "Gemeindeordnungen" heading + 9 cards, then "Stadtrechte" heading + 14 cards
- Same footer

**Key constraints:**
- Use text-gruene-dark (not text-gruene-green) for all body text and links per WCAG (gruene-green fails 4.5:1 contrast)
- Use gruene-green only for decorative accents, large headings (24px+), borders, card accents
- System font stack (no custom fonts)
- All link text must use hover:underline, color gruene-dark
- Header height ~4rem, set scroll-margin-top: 5rem on paragraph articles to account for sticky header
- The CSS link path is `../css/main.css` for law pages (in subdirectory) and `css/main.css` for index
- JS script path is `../js/main.js` for law pages and `js/main.js` for index
