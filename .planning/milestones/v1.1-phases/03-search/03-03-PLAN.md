**A. Update `e2e/playwright.config.js`:**

Change the `webServer.command` to include Pagefind indexing:
```javascript
webServer: {
  command: 'npm run build && npx pagefind --site dist --force-language de && npm run preview -- --port 4173',
  url: 'http://localhost:4173/gemeindeordnung/src/index.html',
  reuseExistingServer: !process.env.CI,
  timeout: 60000, // Increase timeout since build+index takes longer
},
```
This ensures Pagefind index exists when preview server starts. Without it, search tests will fail with import errors.

**B. Create `e2e/tests/search.spec.js` -- Core search tests:**

Tests (all run in desktop-chromium):
1. **SUCH-01: Full-text search returns results** -- Go to index page, type "Gemeinderat" in `#search-input`, wait for `#search-dropdown` to be visible, assert at least 1 `.search-result-item` exists. Screenshot: `e2e/screenshots/search-results.png`

2. **SUCH-02: Search terms highlighted in results** -- After searching "Gemeinderat", check that `.search-result-item mark` elements exist (Pagefind wraps matches in `<mark>`).

3. **SUCH-03: Contextual snippets shown** -- After searching, verify each `.search-result-item` contains excerpt text (not empty), and the excerpt contains text around the match (more than just the search term).

4. **SUCH-06: Result count displayed** -- After searching "Gemeinderat", check `.search-count` text matches pattern like "X Treffer" where X > 0. Screenshot: `e2e/screenshots/search-count.png`

5. **SUCH-07: Empty state shown** -- Type "xyznonexistent123" in search, verify the dropdown shows empty state text containing "Keine Treffer". Verify suggestions are present ("Versuchen Sie einen anderen Suchbegriff"). Screenshot: `e2e/screenshots/search-empty.png`

6. **Min 3 chars hint** -- Type "ab" (2 chars) in search, verify dropdown shows hint about minimum characters. Type one more char: results or empty state appears.

7. **Keyboard shortcut** -- Press "/" key on page, verify `#search-input` is focused. Press Escape, verify dropdown is hidden.

**C. Create `e2e/tests/search-filter.spec.js` -- Filter tests:**

1. **SUCH-04: Bundesland selection persistent** -- Go to index page, click a BL filter chip (e.g., "Wien"), verify chip becomes active. Navigate to a law page, verify the same BL is still selected in search chips (read from LocalStorage or check chip state). Screenshot: `e2e/screenshots/search-filter-active.png`

2. **SUCH-05: Search defaults to BL, toggle all** -- With Wien selected, search "Gemeinderat", verify results count header says "X Treffer in Wien". Click "Alle Bundeslaender" chip, verify results update and count says "X Treffer" (without BL qualifier). Verify more results appear when searching all. Screenshot: `e2e/screenshots/search-filter-all.png`

**D. Create `e2e/tests/search-highlight.spec.js` -- On-page highlighting:**

1. Search "Gemeinderat" on index page, click first result, verify navigation to a law page with `?highlight=Gemeinderat` in URL, verify `.pagefind-highlight` elements exist on the target page. Screenshot: `e2e/screenshots/search-highlight-target.png`

**E. Create `e2e/tests/search-mobile.spec.js` -- Mobile search:**

1. Using desktop-chromium project with explicit viewport resize to 375x812 (same pattern as Phase 02.1):
   - Verify search icon button is visible (full search input is hidden on mobile)
   - Click search icon, verify fullscreen overlay appears
   - Type "Gemeinderat" in overlay search input, verify results appear
   - Screenshot: `e2e/screenshots/search-mobile-overlay.png`
   - Close overlay (Escape or close button), verify overlay is hidden

**Important test patterns from Phase 02.1 decisions:**
- Use `./relative` goto paths in Playwright (e.g., `page.goto('./')` for index, `page.goto('./gemeindeordnungen/wien.html')` for law pages)
- Mobile test uses desktop-chromium project with `page.setViewportSize({ width: 375, height: 812 })`
- Screenshots saved to `e2e/screenshots/` directory
- Wait for Pagefind to load before interacting: after page load, wait for `#search-input` to be visible and enabled, optionally wait a short time for WASM init
