**Step 1: Extract shared prompt constants.** Pull the gendering instructions block (lines 539-554 of the current prompt) into a module-level `const GENDERING_INSTRUCTIONS = ...` string. Pull the FAQ-specific style instructions (no BL-specific claims, citation format, umlaut rules) into a `const FAQ_STYLE_INSTRUCTIONS = ...` string. These will be included in every per-topic prompt.

**Step 2: Add helper functions** (NOT exported, before generateFAQ):
- `loadCuratedTopics(rootDir)`: Reads and parses `data/llm/faq/curated-topics.json`. Throws if file missing.
- `collectParagraphsForTopic(curatedTopic, rootDir)`: Iterates all summary files, collects paragraphs whose topic labels contain any keyword from the curated topic's slug (split on hyphens), title (split on whitespace), or description (words > 4 chars). - `buildTopicPrompt(curatedTopic, paragraphRefs)`: Builds the per-topic prompt string. 1. Load curated topics via `loadCuratedTopics(rootDir)`
2. Determine which topics to generate: if `options.topic` is set, filter to that slug only; otherwise all topics
3. If `options.force`, clean `data/llm/faq/topics/` directory and `topics.json`
4. For each curated topic:
   a. Collect matching paragraphs via `collectParagraphsForTopic()`
   b. Log match count (warn if 0 matches)
   c. Build prompt via `buildTopicPrompt()`
   d. Call `callClaude(prompt)` (existing function, no changes)
   e. Write per-topic result to `data/llm/faq/topics/{slug}.json`
5. Aggregate: read all per-topic JSON files from `data/llm/faq/topics/`, merge into `{ meta: { generatedAt, topicCount }, topics: [...] }`, write to `data/llm/faq/topics.json`
6. Return the aggregated result (same return type as before)
