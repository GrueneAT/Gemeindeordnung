    - Test 3: parseLaw() preserves Originaltext 1:1 (no content modification)
    - Test 4: parseLaw() produces valid JSON matching the expected schema (meta + struktur)
    - Test 5: parseLaw() throws error when HTML has no recognizable paragraphs (fail-fast)
    - Test 6: parseLaw() extracts Absaetze (numbered sub-paragraphs) correctly

    - Test 8: parseAll() writes JSON files to correct category directories (gemeindeordnungen/ vs stadtrechte/)

  <action>
1. Create `scripts/parse-laws.js`:
   - Import cheerio and LAWS from config.js
   - `parseLaw(html, lawKey, lawConfig)`:
     - Load HTML with cheerio
     - Extract the law content area (find the main content div -- RIS pages have navigation/header around content)
     - Identify structural hierarchy: look for Hauptstueck/Abschnitt/Teil headings
     - Extract paragraphen: find paragraph markers (e.g., "SS X" or "Paragraph X" patterns), extract nummer, titel, full text
     - Split text into Absaetze where numbered "(1)", "(2)" etc. patterns exist
     - Preserve Originaltext 1:1 -- no cleanup, no HTML-to-text conversion beyond stripping tags (keep exact wording)
     - Build meta object: bundesland, kurztitel, gesetzesnummer, abfrage, kategorie, stadt, fetchedAt (ISO timestamp), sourceUrl, contentHash (SHA-256 of raw HTML)
     - Validate: throw if paragraph count is 0, throw if no structural elements found
     - Return structured JSON object matching the schema from RESEARCH.md
   - `parseAll()`:
     - Read raw HTML files from `data/raw/gemeindeordnungen/` and `data/raw/stadtrechte/`
     - For each, call parseLaw()
     - Write output to `data/parsed/gemeindeordnungen/{key}.json` and `data/parsed/stadtrechte/{key}.json`
     - Log summary: "Parsed {name}: {N} Paragraphen in {M} Abschnitte"
     - On ANY parse error: throw immediately (fail-fast per user decision)
   - Export `parseLaw` for testing
   - When run directly: call parseAll()

2. The parser MUST be adaptive to structural variation:
   - Some laws use Hauptstuecke > Abschnitte > Paragraphen
   - Some use Abschnitte > Paragraphen (no Hauptstuecke)
   - Some may use Teile or other groupings
   - Parser should detect what exists and build the struktur array accordingly
   - Do NOT hardcode structure per Bundesland unless truly necessary -- prefer a generic approach first

3. Create `tests/parse-laws.test.js`:
   - Import parseLaw from parse-laws.js
   - Load fixture HTML files
   - Test each behavior listed above
   - Use `describe` blocks per Bundesland fixture

4. Add npm script: `"parse": "node scripts/parse-laws.js"` to package.json
